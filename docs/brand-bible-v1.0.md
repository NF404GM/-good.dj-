# The good. Company Brand Bible

## Living Operating System — Version 1.0 (March 2026)

This is the single source of truth for every decision we make—now and for every future instrument we ever ship, whether it’s a script tool, DAW, DJ rig, camera app, note-taker, or anything else that fits the DNA. It is principle-first, not product-first. It is designed to scale infinitely while never drifting. It is the filter every screen, feature name, line of copy, motion, color, icon, and strategic choice must pass through. Refer to it before you design, code, write, name, or pitch—always.

## 0) The One-Sentence Definition

The good. company builds essentialist, pro-grade creative tools that feel like well-crafted instruments—software that gets out of the way, moves beautifully, and makes the creator better.

## 1) Company DNA

### 1.1 What we believe (foundational worldview)

- **The tool shapes the work.** If the tool feels powerful, the creator performs better.  
- **Craft beats novelty.** The goal is not “more features.” The goal is inevitable features.  
- **Software should feel physical.** Not skeuomorphic leather textures—instrument logic: clear states, tactile feedback, spatial memory.  
- **Quiet confidence wins.** We don’t sell hype. We ship proof.

### 1.2 What we’re fighting against (the enemy)

Modern creative software is often:

- Bloated  
- Loud / visually noisy  
- Trend-chasing  
- Loaded with “AI theater”  
- Built for growth metrics over creative flow

Our stance: strip it down until it’s good, then make it better through craft.

### 1.3 What we’re building (the category)

Not “apps.” A browser-native creative instrument suite. Think: hardware-level focus with cloud-level convenience.

## 2) Brand Architecture (two official identities, one spine)

We operate in two valid identity modes that share the exact same principles but express differently.

### 2.1 Identity Mode A: GOOD.[PRODUCT] (uppercase)

- Feels like: modern, editorial, minimalist tech-lux  
- The dot namespace is structural: GOOD.SCRIPT  
- Visual mark: italic bold “G” in a rounded square  
- Voice: confident colleague, very clean, no buzzwords  
- Great for: products that feel like “platforms” (SCRIPT, CUT, CAST)

### 2.2 Identity Mode B: good.[PRODUCT] (lowercase)

- Feels like: studio hardware / utilitarian pro tools  
- Parent: The good. company  
- Suite framing: good.SUITE  
- Wordmark: good. in Inter Bold/ExtraBold, tight tracking  
- Product lockups: good.DAW, good.DJ, good.STUDIO  
- Great for: performance/production tools that should feel like equipment

### The rule that keeps both coherent

Whether we use GOOD. or good., the principles never change:
*Essentialism • Creator-first • Invisible complexity • Quiet confidence • Motion with mass • Tool disappears; output shines*

## 3) Brand Philosophy (the doctrine)

### 3.1 Brand promise

GOOD tools don’t get in your way. They get out of the way—and when they move, they move beautifully.

### 3.2 Core values (operational definitions)

- Craft over features: every addition must feel unavoidable  
- Invisible complexity: power without cognitive load  
- Creator-first: defaults serve creation, not configuration  
- Quiet confidence: no gimmicks, no shouting, no hype  
- Warmth in technology (good. mode): modern tools shouldn’t feel cold—tactility + comfort matter  
- Unpretentious mastery: pro-grade, approachable, no jargon armor  
- Precision craftsmanship: tiny details = trust

### 3.3 The “Good → Better” engine

Your moral (“good could be better”) becomes the company loop:
Strip to essentials → Make essentials feel incredible → Add only what protects flow or increases output quality → Repeat

## 4) The Product Ecosystem (what the company actually is)

### 4.1 The suite map (present + future)

#### Core suite

- GOOD.SCRIPT / good.SCRIPT — write → refine → deliver (teleprompter + writing + rehearsal)  
- good.DAW / good.STUDIO — browser-native music creation + arrangement  
- good.DJ / GOOD.DJ — live performance, mixing, stems, control

#### Expansion orbit

- GOOD.CUT / good.CUT — script-to-timeline video editing  
- GOOD.CAST — podcast + VO production  
- GOOD.TYPE — copywriting assistant for marketing  
- good.DESIGN — vector/raster manipulation  
- good.CODE — minimal collaborative editor  
- good.CLOUD — storage, sharing, collaboration, identity

(and any future instrument that passes the Brand Fit Test in Section 17)

### 4.2 The connective tissue (why it’s a company, not products)

Everything is built around sessions and artifacts:

- **Session**: what you’re doing now (script draft, DJ set, DAW project, photo shoot, etc.)  
- **Artifact**: what you export (final script, stems, video timeline, RAW files, etc.)

The suite advantage: one account, one library, shared components, shared muscle-memory.

## 5) Brand Voice System (how we speak everywhere)

### 5.1 Global voice

Direct • Specific • Calm confidence • Helpful (not patronizing) • Inspiring (output-focused)

### 5.2 Copy rules

**DO:**

- “Start typing.”
- “Synced.”
- “REC.”
- “Glue your mix together.”

**DON’T:**

- “Welcome to your creative journey!”
- “Oops!”
- “Powered by cutting-edge AI”
- Excess exclamation points

### 5.3 Mood-based voice (GOOD.SCRIPT example)

- Creator: warm, slightly poetic (“Flow state engaged.”)  
- Speech: commanding broadcast clarity (“Speak with conviction.”)  
- Studio: technical system tone (“System ready.”)  
- E-Ink: minimal literary (“Focus on the words.”)

(The same calibrated tone scales to any product.)

## 6) Visual Identity System

### 6.1 Logo & marks (GOOD mode)

- G mark: italic bold “G” in rounded square  
- Black/white only  
- Clear space: 0.5× mark width  
- Min size: 24×24px digital  
- Lockup: Mark + GOOD.SCRIPT (default); mark alone for app icons/favicons

### 6.2 Wordmark (good. mode)

- good. in Inter Bold/ExtraBold, tight tracking (-0.05em)  
- Always lowercase, period included  
- Product lockup: good.DJ (product often uppercase + mono)

## 7) Color Systems (two palettes + strict rules)

### 7.1 GOOD (monochrome editorial)

Brand tokens:

- void #000000
- midnight #09090b
- carbon #18181b
- graphite #27272a
- silver #a1a1aa
- bone #f5f5f5
- pure #ffffff  

Functional colors only:

- emerald #10b981 (synced/success)
- red #ef4444 (destructive/record/live)
- amber #f59e0b (warning/unsaved)
- blue #3b82f6 (info/links/collab)  

*Rule: brand is monochrome; color appears only for meaning.*

### 7.2 good. (hardware-warm palette)

- Base: matte black #0f0f0f, charcoal #1f1f1f  
- Text: off-white #f7f5f3, soft gray #e9e6e2  
- Accents: warm gray #bfb7ad, sand #d6cfc6, olive #8a8b7f  

## 8) Typography System

### 8.1 GOOD mode

- System sans stack (native feel)  
- Hierarchy via weight, spacing, case  
- Wide tracking, uppercase labels  
- Mono micro text for stats/timecode  

### 8.2 good. mode (dual type)

- Inter = human interface and reading  
- Roboto Mono / JetBrains Mono = machine data (BPM, Hz, dB, logs, timecode)  

## 9) Motion & Interaction (signature feel)

### 9.1 GOOD motion (springs)

Presets:

- Snappy 500/35  
- Smooth 300/25  
- Gentle 200/20  

Signature: layoutId morph pill, traveling active indicator, blur-to-sharp sidebar.

### 9.2 good. micro-interactions (hardware)

- Under 200 ms, subtle but unmistakable  
- “Zero latency illusion” for critical actions (Play/REC)  
- 1px borders like faceplates (white/10)  

## 10) UI/UX Laws (non-negotiables)

### 10.1 The “instrument rules”

- Spatial memory: panels stay where you put them  
- No hidden primary controls: avoid hamburger for core actions  
- Clear state language  

### 10.2 Contrast discipline

- High contrast only for primary actions  
- structural UI stays low contrast  

### 10.3 Performance rules

- No long loaders  
- Avoid animations on high-frequency UI

## 11) Feature Naming System

Features are verbs or plain nouns. No ™ energy.
Examples: Character Bible, Plot Stitcher, Smart Pace.

## 17) Brand Extension Framework (Section 17)

Infinite scaling for any future instrument.

### 17.1 Brand Fit Test

1. Strip it to essentials in <60 seconds?
2. Feel like a physical instrument?
3. Removing features makes it better?
4. Using only existing systems?
5. Sounds like us?
6. Motion translates?
7. Naming rules followed?

### 17.2 Product Naming

- Always GOOD.[NOUN/VERB] or good.[NOUN/VERB].
- Never hype.

### 17.3 Mode Choice

- GOOD. mode for creation/delivery.
- good. mode for performance/tactile tools.

### 17.4 Visual adaptation

Core palettes and typography never change. Domain signals only for literal replacement.

### 17.5 One-Account rule

Shared library, shared account, shared session concept.
